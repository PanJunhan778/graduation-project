### 1. 公司表 (`company`)
**定位**：系统的基石，用于多租户隔离，同时存储企业的基础画像（纳税人性质、业务描述），作为 AI 诊断全局上下文。

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |
| :--- | :--- | :--- | :--- |
| `id` | bigint | 主键, 自动递增 | 主键ID |
| `name` | varchar(100) | | 公司名称 |
| `description` | text | | **公司描述** (AI全局Prompt上下文，可通过HITL机制更新) |
| `industry` | varchar(50) | | 所属行业 |
| `company_code` | varchar(6) | 唯一索引 `idx_company_code` | **6位唯一企业码** (用于员工注册和绑定公司) |
| `taxpayer_type` | varchar(20) | | **纳税人性质** (如: 小规模纳税人/一般纳税人/个体户。供AI税务诊断参考) |
| `status` | tinyint | 默认 1 | **企业状态** (1启用 0禁用) |
| `created_time` | datetime | | 创建时间 |
| `updated_time` | datetime | | 更新时间 |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 (0正常 1删除) |

### 2. 用户表 (`user`)

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |

| `id` | bigint | 主键, 自动递增 | 主键ID |

| `password` | varchar(255) | | 登录密码 (加密后存储) |
| `role` | varchar(20) | | **系统角色** (枚举: admin / owner / staff) |
| `real_name` | varchar(50) | | 真实姓名 |
| `status` | tinyint | 默认 1 | 账号状态 (1启用 0禁用) |
| `created_time` | datetime | | 账号创建时间 |
| `updated_time` | datetime | | 账号更新时间 |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 |

### 3. 财务流水表 (`finance_record`)
**定位**：系统最核心的业务表，记录每一笔真实的资金流入和流出。

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |
| :--- | :--- | :--- | :--- |
| `id` | bigint | 主键, 自动递增 | 主键ID |
| `company_id` | bigint | 普通索引 `idx_company_id` | 公司ID (实现数据隔离) |
| `type` | varchar(20) | | **收支类型** (枚举: income 收入 / expense 支出) |
| `amount` | decimal(15,2)| 必须 > 0 | **交易金额** (采用15,2高精度，严禁零值与负数) |
| `category` | varchar(50) | 普通索引 `idx_category` | **财务分类** (如: 采购、人工、营销。用于AI按分类汇总) |
| `project` | varchar(100)| | 关联项目 (选填) |
| `date` | date | 普通索引 `idx_date` | 业务发生日期 |
| `remark` | varchar(255) | | **备注** (AI抓取业务细节的重要依据) |
| `created_by` | bigint | | 数据录入人ID |
| `created_time` | datetime | | 数据创建时间 |
| `updated_by` | bigint | | 最后修改人ID |
| `updated_time` | datetime | | 数据更新时间 |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 |

### 5. 税费表 (`tax_record`)
**定位**：只做会计核算结果的录入，不做复杂运算。用于给老板展示真实的税务流出。

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |
| :--- | :--- | :--- | :--- |
| `id` | bigint | 主键, 自动递增 | 主键ID |
| `company_id` | bigint | 联合索引1 `idx_company_period_type` | 公司ID |
| `tax_period` | varchar(20) | 联合索引2 `idx_company_period_type` | **税款所属期** (枚举: `2026-08`, `2026-Q3`, `2026-Annual`) |
| `tax_type` | varchar(50) | 联合索引3 `idx_company_period_type` | **具体税种** (如: 增值税, 企业所得税) |
| `declaration_type`| varchar(20) | | **申报类型** (枚举: 日常/预缴, 年度汇算清缴) |
| `tax_amount` | decimal(15,2)| | **税额** (允许为负数，代表汇算清缴时的退税) |
| `payment_status`| tinyint | | **缴纳状态** (`0`: 待缴纳, `1`: 已缴纳, `2`: 零申报/免征) |
| `payment_date` | date | 允许为空 | 实际打款日期 (状态为`已缴纳`时填写) |
| `remark` | varchar(255) | | 备注 (供会计向老板解释特殊情况，如免税额度) |
| `created_by` | bigint | | 录入人ID |
| `created_time` | datetime | | 创建时间 |
| `updated_by` | bigint | | 最后修改人ID |
| `updated_time` | datetime | | 更新时间 |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 |

| `user_id` | bigint | | 发起修改的操作人ID |
| `module` | varchar(50) | | **被操作模块** (finance / employee / tax等) |
| `operation_type`| varchar(20) | | **操作类型** (如 UPDATE, 逻辑 DELETE) |
| `target_id` | bigint | | 被操作数据的主键ID |
| `field_name` | varchar(50) | | **发生变更的字段名** (如 `amount`, `salary`) |
| `old_value` | text | | 修改前的值 |
| `new_value` | text | | 修改后的值 |

### 7. AI 会话记录表 (`ai_chat_log`)
**定位**：持久化大模型交互记录，用于前端瀑布流展示与后端的“滑动窗口记忆”上下文提取。

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |
| :--- | :--- | :--- | :--- |
| `id` | bigint | 主键, 自动递增 | 主键ID |
| `company_id` | bigint | 普通索引 `idx_company_id` | 公司ID |
| `user_id` | bigint | | **发起对话的用户ID** (关联 `user` 表，追溯具体操作人) |
| `session_id` | varchar(36) | 普通索引 `idx_session_id` | **会话标识** (UUID，区分不同对话轮次，前端"新对话"时生成) |
| `role` | varchar(20) | | **消息发送方** (枚举: `user` 用户 / `assistant` 大模型 / `system` 系统机制) |
| `message_type` | varchar(32) | 默认 `text` | **消息类型** (`text` / `markdown` / `action_required` / `action_result`，用于区分普通回复与待确认动作卡片) |
| `content` | longtext | | **消息内容** (聊天文本或函数调用的JSON结果) |
| `metadata_json` | longtext | 允许为空 | **元数据JSON** (存储动作卡片、结构化上下文等附加信息) |
| `create_time` | datetime | 普通索引 `idx_create_time`| 消息生成时间 (用于前端按序拉取历史记录) |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 |

### 8. AI 待确认动作表 (`ai_pending_action`)
**定位**：承接 AI 辅助决策中的 HITL（Human in the Loop）确认流程。当 AI 准备修改关键企业信息时，先生成待确认动作，由用户审核通过后再落库，避免模型直接改写核心数据。

| 字段名 | 数据类型 | 约束/索引 | 字段说明 (核心业务逻辑) |
| :--- | :--- | :--- | :--- |
| `id` | bigint | 主键, 自动递增 | 主键ID |
| `company_id` | bigint | 联合索引1 `idx_company_status` | 公司ID |
| `user_id` | bigint | | **发起动作的用户ID** (记录是谁触发了 AI 辅助修改请求) |
| `session_id` | varchar(36) | 普通索引 `idx_session_id` | **会话标识** (关联 AI 对话上下文，便于前后端追踪同一轮交互) |
| `chat_message_id` | bigint | 允许为空 | **关联卡片消息ID** (指向需要用户确认的那条会话消息) |
| `action_type` | varchar(64) | | **动作类型** (标识本次待确认操作的业务类别) |
| `confirm_token` | varchar(64) | 唯一索引 `uk_confirm_token` | **确认令牌** (用于提交批准/拒绝操作时做安全校验) |
| `old_value` | longtext | 允许为空 | 修改前的旧值快照 |
| `proposed_value` | longtext | | **拟更新值** (AI 生成、等待用户确认的新值内容) |
| `status` | varchar(20) | 联合索引2 `idx_company_status` | **处理状态** (`pending` / `approved` / `rejected` / `expired`) |
| `expires_at` | datetime | | **过期时间** (超时后动作自动失效，避免旧确认长期可用) |
| `processed_by` | bigint | 允许为空 | 最终处理人ID |
| `processed_at` | datetime | 允许为空 | 最终处理时间 |
| `created_time` | datetime | | 创建时间 |
| `updated_time` | datetime | | 更新时间 |
| `is_deleted` | tinyint | 默认 0 | 逻辑删除 |
